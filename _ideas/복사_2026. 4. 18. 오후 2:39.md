# 맥에서 MP3 여러 개를 아이폰 오디오북으로 만드는 법 (챕터 포함)

> **Books 앱에서 챕터까지 딱딱 나뉘는 진짜 오디오북 만들기**  
> 터미널 초보도 따라할 수 있게, 명령어 하나씩 차근차근.

---

## 왜 M4B인가?

아이폰 Books 앱은 `.m4b` 포맷을 오디오북으로 인식한다.  
MP3 파일 여러 개를 그냥 복사하면 "음악"으로 취급되거나 제각각 따로 열린다.  
이게 되게 불편한게 15초 앞이나 뒤로 갈 수 가 없다. 이것 처럼 짜증나는 일도 없다.
M4B로 묶으면:

- Books 앱에서 **하나의 오디오북**으로 인식
- **챕터별 이동** 가능 (각 MP3가 챕터로 분리됨)
- 청취 위치 자동 저장

---

## 준비물

- Mac
- MP3 파일들 (한 폴더에 모아두기)
- 터미널 (기본 내장 앱)
- 약 15~30분 (설치 포함)

---

## STEP 1. Homebrew 설치

Homebrew는 맥에서 개발 도구를 쉽게 설치할 수 있게 해주는 패키지 매니저다.  
이미 설치되어 있다면 이 단계는 건너뛰어도 된다.

터미널을 열고 아래 명령어를 붙여넣는다:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

**진행 중 주의사항:**
- 비밀번호 입력 요청 → 맥 로그인 비밀번호 입력 (입력해도 화면에 표시 안 됨, 정상)
- `Press RETURN/ENTER to continue` → 엔터키 누르기
- macOS 버전 경고가 떠도 무시하고 기다리면 됨
- 설치에 5~15분 걸릴 수 있음 (openssl 등 빌드 중)
- 터미널에 `%` 프롬프트가 다시 나타나면 완료

---

## STEP 2. FFmpeg 설치

FFmpeg는 오디오/영상 변환의 핵심 도구다.

```bash
brew install ffmpeg
```

완료 후 `%` 프롬프트가 나타나면 성공이다.

---

## STEP 3. MP3 파일 준비

변환할 MP3 파일들을 **한 폴더**에 모아둔다.  
파일 이름 앞에 `01_`, `02_`, `03_` 같은 번호를 붙여두면 챕터 순서가 자동으로 맞춰진다.

예시:
```
01_Introduction.mp3
02_Chapter1.mp3
03_Chapter2.mp3
...
```

> ⚠️ **주의:** 파일명에 `'` (apostrophe, 작은따옴표)가 있으면 오류가 날 수 있다.  
> 미리 파일명에서 `'` 를 제거하거나 다른 문자로 바꿔두는 것을 권장한다.

---

## STEP 4. 터미널에서 폴더로 이동

```bash
cd ~/Desktop/폴더이름
```

예시: 폴더 이름이 `DragonMaster-19`라면
```bash
cd ~/Desktop/DragonMaster-19
```

이동 후 파일 목록을 확인한다:

```bash
ls *.mp3
```

파일들이 쭉 출력되면 정상이다.

---

## STEP 5. 파일 목록 생성

MP3 파일 목록을 정렬된 순서로 저장한다:

```bash
ls *.mp3 | sort > filelist.txt && cat filelist.txt
```

01, 02, 03... 순서로 파일명이 출력되면 성공이다.

---

## STEP 6. FFmpeg용 concat 파일 생성

FFmpeg가 읽을 수 있는 형식으로 목록을 변환한다:

```bash
while IFS= read -r f; do echo "file '$f'"; done < filelist.txt > concat.txt && cat concat.txt
```

아래와 같이 각 파일 앞에 `file '...'` 형식으로 출력되면 성공이다:

```
file '01_Introduction.mp3'
file '02_Chapter1.mp3'
...
```

---

## STEP 7. MP3 전체 합치기 (임시 파일 생성)

모든 MP3를 하나의 M4A 파일로 합친다.  
파일 수에 따라 수 분이 걸릴 수 있다:

```bash
ffmpeg -f concat -safe 0 -i concat.txt -c:a aac -b:a 128k temp_merged.m4a -y
```

완료되면 같은 폴더에 `temp_merged.m4a` 파일이 생성된다.  
아래처럼 `bitrate`와 `speed` 정보가 나오고 `%` 프롬프트가 돌아오면 성공이다:

```
size=  84480kB time=01:42:05.58 bitrate= 113.0kbits/s speed=72.1x
```

---

## STEP 8. 챕터 메타데이터 생성

각 MP3의 길이를 측정해서 챕터 정보를 만든다.  
아래 명령어를 **전체 복사해서 한 번에** 붙여넣는다:

```bash
echo ";FFMETADATA1" > chapters.txt
start=0
while IFS= read -r f; do
  duration=$(ffprobe -v error -show_entries format=duration \
    -of default=noprint_wrappers=1:nokey=1 "$f")
  duration_ms=$(echo "$duration * 1000 / 1" | bc)
  end=$((start + duration_ms))
  title="${f%.mp3}"
  echo "" >> chapters.txt
  echo "[CHAPTER]" >> chapters.txt
  echo "TIMEBASE=1/1000" >> chapters.txt
  echo "START=$start" >> chapters.txt
  echo "END=$end" >> chapters.txt
  echo "title=$title" >> chapters.txt
  start=$end
done < <(ls *.mp3 | sort)
```

에러 없이 `%` 프롬프트가 돌아오면 성공이다.

내용을 확인하려면:
```bash
cat chapters.txt
```

챕터 정보들이 쭉 출력되면 정상이다.

---

## STEP 9. 최종 M4B 파일 생성

합친 오디오와 챕터 정보를 결합해서 최종 오디오북 파일을 만든다:

```bash
ffmpeg -i temp_merged.m4a -i chapters.txt \
  -map_metadata 1 -codec copy output.m4b -y
```

완료되면 `output.m4b` 파일이 생성된다. 🎉

---

## STEP 10. 임시 파일 정리

작업 중 생성된 임시 파일들을 삭제한다:

```bash
rm -f temp_merged.m4a filelist.txt concat.txt chapters.txt
```

---

## STEP 11. 아이폰으로 전송

1. USB 케이블로 아이폰과 맥을 연결한다
2. Finder 열기 → 사이드바에서 아이폰 클릭
3. 상단 탭에서 **Books** 선택
4. `output.m4b` 파일을 Books 영역으로 **드래그 앤 드롭**
5. 아이폰 Books 앱에서 확인!

## 전체 명령어 요약 (복붙용)

폴더 이름만 바꿔서 순서대로 실행하면 된다.

```bash
# 0. 폴더로 이동 (폴더 이름 바꿀 것!)
cd ~/Desktop/폴더이름

# 1. 파일 목록 생성
ls *.mp3 | sort > filelist.txt

# 2. concat 파일 생성
while IFS= read -r f; do echo "file '$f'"; done < filelist.txt > concat.txt

# 3. MP3 합치기 (시간 걸림)
ffmpeg -f concat -safe 0 -i concat.txt -c:a aac -b:a 128k temp_merged.m4a -y

# 4. 챕터 메타데이터 생성
echo ";FFMETADATA1" > chapters.txt
start=0
while IFS= read -r f; do
  duration=$(ffprobe -v error -show_entries format=duration \
    -of default=noprint_wrappers=1:nokey=1 "$f")
  duration_ms=$(echo "$duration * 1000 / 1" | bc)
  end=$((start + duration_ms))
  title="${f%.mp3}"
  echo "" >> chapters.txt
  echo "[CHAPTER]" >> chapters.txt
  echo "TIMEBASE=1/1000" >> chapters.txt
  echo "START=$start" >> chapters.txt
  echo "END=$end" >> chapters.txt
  echo "title=$title" >> chapters.txt
  start=$end
done < <(ls *.mp3 | sort)

# 5. 최종 M4B 생성
ffmpeg -i temp_merged.m4a -i chapters.txt \
  -map_metadata 1 -codec copy output.m4b -y

# 6. 임시 파일 정리
rm -f temp_merged.m4a filelist.txt concat.txt chapters.txt

echo "✅ 완료! output.m4b 파일을 아이폰 Books 앱으로 전송하세요."
```

## 자주 묻는 질문

**Q. `mapfile: command not found` 에러가 나요.**  
A. 맥의 기본 쉘은 zsh인데, `mapfile`은 bash 전용 명령어라 안 된다. 이 가이드의 명령어는 zsh에서도 동작하도록 작성되어 있으니 위 명령어를 그대로 사용하면 된다.

**Q. 파일명에 공백이 있어도 되나요?**  
A. 된다. `while IFS= read -r` 방식이 공백 있는 파일명도 올바르게 처리한다.

**Q. 파일명에 `'`(작은따옴표)가 있으면 어떻게 되나요?**  
A. 오류가 발생하면서 해당 파일부터 처리가 중단된다. 작업 전에 파일명의 `'`를 제거하거나 다른 문자로 바꿔두는 것을 권장한다.

**Q. 챕터 이름을 바꾸고 싶어요.**  
A. MP3 파일명이 곧 챕터명이 된다. 파일명을 원하는 이름으로 미리 바꿔두면 된다.

**Q. output.m4b 파일명을 바꾸고 싶어요.**  
A. 명령어에서 `output.m4b` 부분을 원하는 이름으로 바꾸면 된다.  
예: `DragonMaster19.m4b`
