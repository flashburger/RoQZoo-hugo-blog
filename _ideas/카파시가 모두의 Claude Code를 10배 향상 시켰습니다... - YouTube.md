[카파시가 모두의 Claude Code를 10배 향상 시켰습니다... - YouTube](https://www.youtube.com/watch?v=H9Wml5xDLLY)


이 영상은 안드레이 카파시(Andrej Karpathy)가 제안한 'LLM 위키(LLM Wiki)' 개념을 활용해, 복잡한 RAG(검색 증강 생성) 기술 없이도 마크다운 파일만으로 자신만의 AI 지식 베이스를 구축하는 방법을 설명합니다.

핵심 개념 및 특징

LLM 위키란?: 벡터 데이터베이스나 별도의 서버 인프라 없이, 로컬 마크다운 폴더를 활용해 지식을 정리하고 관계를 연결하는 지식 관리 시스템입니다(2:14).
구조: 원본 자료를 저장하는 '로우(raw) 폴더'와 정리된 정보를 담는 '위키(wiki) 폴더' 두 가지만으로 충분합니다(2:47).
자동화: Claude Code를 활용해 자료를 입력(ingest)하고, 분류 및 태그 지정, 문서 간 링크 연결을 자동으로 수행할 수 있습니다(6:49).
장점: 지식이 복리처럼 쌓여 세컨드 브레인 역할을 하며, AI 에이전트와 연동하여 필요할 때 즉각적으로 지식을 찾아 답을 얻을 수 있습니다(1:59).
주요 세팅 단계

환경 설정: *Obsidian(옵시디언)*을 설치하여 시각적으로 지식 그래프를 관리하고, Claude Code로 파일 작업을 자동화합니다(4:47).
데이터 저장: Obsidian Web Clipper를 사용하여 웹 자료를 로우 폴더에 자동으로 수집합니다(9:39).
지식 자동화: Claude Code에게 프롬프트를 입력하여 수집된 자료를 분석하고 위키 페이지로 변환하게 합니다(11:06).
유지보수: '린팅(linting)'을 통해 깨진 링크를 정리하거나 '핫 캐시(hot cache)'를 사용하여 토큰 비용을 절감하며 효율적으로 관리합니다(16:45).
이 시스템을 구축하면 반복적인 작업 없이도 자신의 사업, 업무, 학습 자료를 체계적으로 관리하고, AI와 대화하며 지식을 확장해 나갈 수 있습니다.
