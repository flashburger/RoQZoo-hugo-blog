[클로드코드 한도가 자꾸 차는 분들 보세요 - 토큰 절약법 정리 - YouTube](https://www.youtube.com/watch?v=7l7orz_rtcQ)


이 영상은 클로드 코드(Claude Code) 사용 시 발생하는 토큰 한도 문제를 해결하기 위한 구체적인 입출력 토큰 절약 방법을 정리한 가이드입니다.

1. 출력 토큰 관리 (비용이 5배 더 비싼 출력 최적화)

모델 선택: 대부분의 작업은 Sonnet으로 충분하므로 메인 모델을 Sonnet으로 설정하세요 (01:08).
Opus 활용: 복잡한 설계는 Opus Plan 모드를 사용하거나, Advisor 기능을 활용해 어려운 판단 순간에만 Opus에 조언을 구하세요 (01:27-01:50).
확장 사고 조절: /effort 명령어로 사고 과정을 조절해 불필요한 토큰 소비를 줄이세요 (03:16).
Plan 모드: 작업 실행 전 계획서를 먼저 확인하여 헛수고를 방지하세요 (03:40).
2. 입력 토큰 관리 (컨텍스트 관리)

진단: /context를 통해 현재 컨텍스트 사용량을 확인하세요 (04:16).
대화 관리:
작업 전환 시 /clear로 대화창을 비우세요 (06:25).
대화가 길어지면 /compact를 사용해 요약하세요 (06:48).
/btw (관련 없는 질문 시) 또는 /rewind (실수 복구)를 통해 컨텍스트 오염을 막으세요 (08:45-09:14).
프롬프트 캐싱: 5분 TTL 제한을 인지하고 자리 비움 시 컨텍스트 관리에 유의하세요 (09:50).
설정 파일 최적화: CLAUDE.md는 200줄 이내로 유지하고, 사용하지 않는 MCP 도구는 비활성화하세요 (10:39-11:15).
3. 보너스 팁

코덱스 플러그인 활용: 클로드 한도가 부족할 경우 오픈AI의 코덱스를 연동하여 병행 사용하면 토큰 여유를 확보할 수 있습니다 (12:12).
